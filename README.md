### 📍 Commit Message Convention

|    Type     | Description  |
|:-----------:|---|
|   `Feat:`   | 새로운 기능 추가 |
|   `Fix:`    | 버그 수정 |
| `Refactor:` | 코드 리팩토링 |

### 💡 규칙 💡
- 새로운 기능을 구현한 후에는 Feature/~ branch를 만들어 git에 올린다
- Feature/~ branch에 있는 코드는 develop branch에 pull request를 보낸다
- develop branch에서 병합 후 최종적으로 main branch에서 배포 예정
